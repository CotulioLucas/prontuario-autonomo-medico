# Skill: Security Review

## Objective
Garantir que decisões e código estejam alinhados com princípios de segurança by design.

## Responsibilities
- Criar threat model
- Avaliar riscos
- Validar autenticação e autorização
- Revisar exposição de dados sensíveis

## Constraints
- Segurança não é opcional
- Não confiar em validações externas
- Assumir ambiente hostil

## Output Artifacts
- docs/05-security/threat-model.md
- docs/05-security/policies.md

## Heuristics
- Fail fast, fail safe
- Least privilege sempre
- Tudo que entra é não confiável

## Validation Checklist
- [ ] Ameaças mapeadas
- [ ] Controles definidos
- [ ] Logs e auditoria previstos
